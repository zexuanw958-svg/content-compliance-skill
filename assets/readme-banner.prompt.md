# README 封面图生成提示词

用于 GPT 图片生成。建议生成 `1200x360` 的横幅图，保存为 `assets/readme-banner.png`。

```text
请生成一张 1200x360 的 GitHub README 横幅封面图，主题是“内容合规检测 Skill”，面向中国自媒体创作者，用于抖音和小红书内容发布前的合规风险检测。

画面风格：
- 高级、专业、现代，像成熟 SaaS 工具的产品封面。
- 深色背景，整体干净、有科技感，但不要廉价霓虹，不要花哨渐变。
- 可以出现抽象的短视频发布界面、手机内容预览、检测报告仪表盘、10 格风险条、安全分、风险提示卡片。
- 配色以深蓝黑、白色文字为主，点缀青色、红色、粉色；风险条可以使用绿色、黄色、红色。
- 不要使用抖音、小红书或任何平台官方 logo。
- 不要水印，不要多余英文，不要乱码，不要错别字。
- 左侧主标题区域要清晰，右侧可以是仪表盘和短视频内容预览。

画面中必须出现这些简体中文文字，并且要求文字清晰、漂亮、无错别字：
主标题：内容合规检测 Skill
副标题：抖音 / 小红书发布前风险体检
辅助文案：选题前、成稿后、投放前都能用
右侧仪表盘：整体安全分 8/10
右侧风险提示：最弱风险点：评论区拿链接

排版要求：
- 主标题必须是第一视觉焦点。
- 中文字体要现代、干净、像专业产品官网，不要像系统默认字体，也不要像儿童海报。
- 文字不要贴边，不要拥挤，不要和仪表盘元素重叠。
- 右侧仪表盘只作为辅助信息，不要抢走主标题。
- 适合放在 GitHub README 顶部，缩小后仍能看清主题。
```
